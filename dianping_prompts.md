## 一、整体架构设计提示词

```
作为资深全栈架构师，基于SpringBoot+Vue技术栈设计类大众点评系统的分布式架构，核心要求如下：
1. 核心目标：支撑大数据个性化推荐、大模型全场景集成、高并发（每秒千级订单/万级用户访问）；
2. 架构拆分：按用户端（小程序）、商户端、管理员端拆分前端，按推荐/订单/用户/大模型模块拆分后端微服务；
3. 技术栈落地：整合MySQL+Redis+Kafka，明确各组件的核心作用（如Redis缓存推荐结果、Kafka削峰订单请求）；
4. 输出要求：
   - 画出架构拓扑图（文字版），标注各模块交互逻辑；
   - 说明分布式架构的高可用设计（熔断降级、数据分片、负载均衡）；
   - 明确大模型（LLM）与业务系统的对接方式（轻量化部署+接口适配）；
   - 给出推荐算法（协同过滤+用户画像）的工程化落地路径。
```

## 二、后端核心模块开发提示词

### 1. 个性化推荐模块（SpringBoot）

```
基于SpringBoot开发类大众点评系统的个性化推荐核心模块，要求：
1. 核心能力：
   - 采集用户行为数据（浏览/收藏/消费/评价）、商户标签数据，构建用户画像；
   - 实现协同过滤算法+场景匹配算法（如“带老人聚餐”场景推荐清淡商户）；
   - 推荐结果通过Redis缓存，缓存预热/失效策略适配高并发；
2. 接口设计：
   - 提供“为你推荐”列表接口（支持按位置/场景/新品维度筛选）；
   - 接口响应时间≤100ms，适配万级并发；
3. 数据交互：
   - 同步商户标签、订单数据至推荐算法库；
   - 输出推荐效果统计指标（点击率/转化率）。
请输出完整的代码实现（含实体类/算法逻辑/接口层/缓存层），并标注关键优化点。
```
